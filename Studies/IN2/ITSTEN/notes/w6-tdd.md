# TDD

## Wanneer

Wanneer TDD gebruiken?
- Bij software waarbij alles goed moet gaan, ie banken, ziekenhuizen, etc.

## Waarom
Waarom TDD?
- Forceert dat er maar kleine stukjes code worden geschreven
- Voorkomt bugs
- Maakt het makkelijker om grote codebases te onderhouden
- Het vinden van nieuwe bugs gaat sneller

Waarom niet TDD?
- Vertraagd het development proces 
- Moeilijk voor een junior developer (binnen een codebase / project / bedrijfsprocessen)
- Lastig te onderhouden
