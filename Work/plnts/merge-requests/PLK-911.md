# Feat: Address Book

This MR introduces the following features for the Address Book:

- [x] Edit address
- [x] Delete address
- [x] Set as shipping/billing address
- [ ] Add new address (fixing some issues with the form)

## Changes Made:

- **New Components Added:**
  - `AddressBook`: Displays a list of user addresses.
  - `AddressEntry`: Shows individual address details.
  - `AddressActions`: Provides actions like edit, delete, set as default.
  - `AddAddressButton`: Opens a dialog to add a new address.
  - `ChangeOrAddAddressDialog`: Handles adding or updating addresses.
  - `ChangeAddressDetailsForm`: Form for changing address details.

- **Routing Updates:**
  - Added `/account/my-address-book` page.
  - Moved personal details to `/account/overview`.
  - Removed old address update pages.

- **Translations:**
  - Updated message files (`.json` files) for multiple locales with new keys related to the address book features.

- **API and Services:**
  - Implemented TRPC mutations and queries for:
    - Updating address details.
    - Creating new addresses.
    - Deleting addresses.
  - Updated `MagentoAccountService` to handle address mutations.
  - Added country and region fetching in `MagentoStoreService`.

- **Validation and Error Handling:**
  - Added form validation for address fields using `valibot`.
  - Implemented error messages for invalid inputs (e.g., invalid house number).
  - Displayed TRPC API errors in the UI using `DisplayTrpcApiError`.

- **UI Enhancements:**
  - Added skeleton loaders for the address book.
  - Updated the account menu and footer links to point to the new address book page.
  - Improved dialog components to support dynamic content and actions.
